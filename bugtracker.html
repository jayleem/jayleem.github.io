Bug Tracker

Bug Tracker was a full-stack development project and was intended for use by a single organization. The purpose of this project was to allow software developers to manage bugs and other issues through the use of tickets.

The full website is available here.

Registration requires approval by an administrator.

Technology

  • Cloudfire Store
  • Angular
  • Firebase
  • Some of the JavaScript Libraries used in this project include: RxJS, AngularFire, Charts.JS, etc.

Design Process

The process for this project was pretty standard, I sketched a wireframe for both mobile and desktop versions. I always design with mobile in mind so, the only difference for mobile is the layout of the dashboard. The final website looks very similar to this wireframe.

Authentication

I read through the documentation for Firebase on implementing authentication. I implemented email and password based authentication which aligned with the project requirements.

Problems and Solutions

Problem: Prior to this project I had no knowledge of Firebase or RxJS

Solution: Reading documentation that exists for both Firebase and RxJS .

Problem: High Cloudstore Read Usage

Solution: The problem was that Cloudfire would count reading a single document in a collection as 1 read which meant depending on how many documents were in a collection a single user could easily exceed plan limits. This a pretty serious issue as a single user could easily go through 1-2k reads per hour.

I set out to fix this problem by creating an aggregated collection. I then read through the documentation on how to write Cloud functions for Firebase. The function I ended up writing was pretty simple and would just keep the aggregated collection updated when Firebase detected changes.

After implementing this solution a user could retrieve the aggregated collection which would only count as a single read.
